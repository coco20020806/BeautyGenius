# Makeup Visual Optimization

Skill source directory: `skills/makeup-visual-optimization/`（Beauty Genius 仓库内）。

Attribution：上游来源 [JoyelleZ/makeup — makeup-visual-optimization](https://github.com/JoyelleZ/makeup/tree/main/makeup-visual-optimization)。

Convert a parsed makeup tutorial into an optimized visual-follow-along version. The skill does not parse videos or generate finished makeup previews; it rewrites diagram instructions, visual layer patches, and practice guidance for the Adaptation Service and Visual Tutorial Service.

## When To Use

- A tutorial already has structured steps, products, parts, visual layers, or eye-detail regions.
- The user provides preferences such as style, occasion, retained original modules, target areas, skin type, face shape, makeup goals, or execution limits.
- The task asks for 优化图示, 个性化优化, 微调输入规则, 可视化教程改写, eye-detail adaptation, or checklist patches.

Do not use this skill for raw video parsing, brand-specific product recommendation, medical/skin diagnosis, or identity-changing face generation.

## Inputs

Expected sources:

- `tutorial`: existing tutorial object or relevant `steps[]`, including `visual_layers`, `instruction`, `adaptation_note`, and `video_clip` when available.
- `optimization_input`: the user questionnaire and free text. See [references/input-schema.md](references/input-schema.md).
- Optional `user_profile`: only use fields already provided or explicitly generated by the product. Do not infer sensitive face traits from questionnaire-only input.

## Pipeline

Process tutorial steps in source order.

1. **Load inputs**: read tutorial steps and normalize questionnaire values.
2. **Classify intent**: split inputs into style, occasion, face-state goal, execution limit, and retained original modules.
3. **Select impacted modules**: map target areas to base, brow, eye, cheek, contour, highlight, lip, and overall.
4. **Apply rules**: use [references/optimization-rules.md](references/optimization-rules.md) for placement, opacity, color, technique, and step-order changes.
5. **Patch diagram contract**: output machine-readable `step_adjustments`, `visual_layer_patch`, and `eye_detail_region_patches` following [references/output-contract.md](references/output-contract.md).
6. **Create image prompt only if needed**: when the user needs an image-generation prompt for a fixed base face diagram, use [references/diagram-prompt.md](references/diagram-prompt.md).
7. **Report**: summarize what changed first, then include structured JSON.

## Hard Constraints

- Keep the project's tone: precise, friendly, and non-judgmental.
- Do not say "缺陷", "显老", "脸大", "不适合你", or make attractiveness judgments.
- Keep diagram changes drawable: every note must include position, range, direction, opacity/intensity, or boundary.
- Preserve user-selected retained modules unless a requested face-state goal directly targets that module; in that case, keep the original color/product family and patch only placement, range, intensity, or technique.
- Product substitutions must be by texture/color family/finish keywords, not unsupported brand claims.
- Eye changes must remain on one enlarged eye base image with clickable regions.
- If a face field is unknown, output a low-confidence neutral rule instead of inventing analysis. Skin type only changes operation advice, product texture, layering, and friction control; it must not change visual layer position, shape, or opacity.

## Acceptance Checklist

- [ ] Output states the primary goal and retained original modules.
- [ ] Each changed step has `original`, `adapted`, `actions`, `visual_layer_patch`, and `adaptation_note`.
- [ ] Layer opacity stays inside project ranges from [references/output-contract.md](references/output-contract.md) **Opacity Ranges** (e.g. Base 0.18–0.28, Blush 0.30–0.48).
- [ ] Shortening-midface cases flatten blush placement and lower contour start where relevant.
- [ ] Execution limits change tools or complexity, not only copy.
- [ ] Practice checklist patches are directly followable.
- [ ] User-facing notes are neutral and specific.
- [ ] If diagram prompt is produced, it preserves the fixed base face and asks for range markings, not full makeup rendering.
